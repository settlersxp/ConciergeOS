# ConciergeOS Backend

> FastAPI-based backend for the ConciergeOS hotel management system.

## 📋 Overview

The backend provides a RESTful API for managing hotel reservations, guest search, performance testing, and prompt management. It's built with:

- **FastAPI** — High-performance web framework
- **SQLAlchemy** — SQL ORM for database operations
- **Alembic** — Database migrations
- **OpenAI SDK** — LLM integration for AI-powered guest search
- **APScheduler** — Scheduled task management for prompt groups

## 🚀 Quick Start

### Prerequisites

- Python 3.12+
- [UV](https://github.com/astral-sh/uv) package manager

### Installation

```bash
# From the project root
uv sync
```

### Database Setup

```bash
# Apply all database migrations
cd backend && uv run alembic upgrade head
```

### Generate Initial Data

```bash
# Generate multilingual names (~400 names across 8 writing systems)
uv run python Generator/generate_names.py

# Generate room definitions (~205 rooms)
uv run python Generator/generate_rooms.py

# Populate database with rooms, guests, and reservations
uv run python Generator/populate_reservations.py
```

### Start the Server

```bash
# Development mode (auto-reload on changes)
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

# Production mode
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

The API will be available at `http://localhost:8000`.

## 📖 API Documentation

Interactive documentation is auto-generated by FastAPI:

| Format | URL | Description |
|--------|-----|-------------|
| Swagger UI | `http://localhost:8000/docs` | Interactive docs with try-it-out |
| ReDoc | `http://localhost:8000/redoc` | Alternative documentation |
| OpenAPI JSON | `http://localhost:8000/openapi.json` | Raw OpenAPI spec |

Export the OpenAPI spec to a file:
```bash
uv run python export_openapi.py
```

## 🗄️ Database

SQLite is used by default. The database is auto-created on first run.

### Migrations

```bash
# Create migration after model changes
uv run alembic revision --autogenerate -m "description"

# Apply migrations
uv run alembic upgrade head

# Revert last migration
uv run alembic downgrade -1

# View migration history
uv run alembic history
```

## ⚙️ Configuration

Application settings are stored in `app/config.json` and managed via the `/api/settings` endpoint. Key settings include:

- `llm_base_url` — LLM API endpoint
- `llm_api_key` — API authentication key
- `llm_model` — Default model name
- `openai_api_key` — OpenAI API key
- `openai_model` — OpenAI model name

## 🔌 API Endpoints

### Reservations
- `GET /api/reservations` — Reservations grouped by room
- `POST /api/reservations/shift` — Shift reservation dates

### Guest Search
- `POST /api/guest-search` — AI-powered guest search
- `POST /api/guest-search/extract-name` — Extract name from media

### Settings
- `GET /api/settings` — Get configuration
- `POST /api/settings` — Update configuration
- `GET /api/models` — List available LLM models

### Performance Testing
- `POST /api/performance-testing` — Run benchmarks
- `GET /api/performance-testing/results` — Get test results
- `GET /api/performance-testing/stats` — Aggregated statistics

### Prompts
- `GET /api/prompts` — List prompts
- `POST /api/prompts/ai-improve` — AI-assisted prompt improvement

See the interactive Swagger UI for the complete API reference.
